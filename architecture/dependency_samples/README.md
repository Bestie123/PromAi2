# Dependency Tracking Options

## Option 1: Enhanced JSON Dependencies
**Файл:** `option1_enhanced_json.json`
**Подход:** Расширенная JSON структура с типами зависимостей

### Особенности:
- Hard/soft зависимости
- Blocking критерии
- Простая валидация
- Линейный порядок разработки

### Плюсы:
- Простота реализации
- Понятная структура
- Легко валидировать

### Минусы:
- Нет параллельной разработки
- Ограниченная гибкость

---

## Option 2: Development Phases  
**Файл:** `option2_phases.json`
**Подход:** Фазовая разработка с milestone'ами

### Особенности:
- 4 фазы разработки
- Milestone'ы (MVP, Beta, Release)
- Оценка времени
- Параллельная работа внутри фаз

### Плюсы:
- Четкие этапы
- Планирование времени
- Возможность параллельной работы

### Минусы:
- Жесткие границы фаз
- Сложнее изменить порядок

---

## Option 3: Dependency Graph
**Файл:** `option3_graph.json`  
**Подход:** Граф зависимостей с весами и рисками

### Особенности:
- Веса узлов
- Типы связей (enables/requires)
- Критический путь
- Матрица рисков
- Параллельные треки

### Плюсы:
- Максимальная гибкость
- Оптимизация критического пути
- Управление рисками
- Параллельная разработка

### Минусы:
- Сложность реализации
- Требует алгоритмы планирования

---

## Рекомендация

**Для PromAi PKM:** Option 2 (Phases) + элементы Option 3 (риски)

Причины:
- Четкие milestone'ы для MVP
- Возможность параллельной работы
- Управление рисками
- Простота понимания для команды
